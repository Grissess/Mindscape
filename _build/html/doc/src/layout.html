layout – Layout Manager¶

This module provides an interface to a simple layout manager which is entirely compatible with the scenegraph (provided that it is rendered after 3D geometry and with the depth test turned off). The layout manager is a tiling manager, which means that it deals with Grids.

class layout.LayoutCell(weight=1.0, fixed=0)[source]¶

Bases: object

A LayoutCell represents a single cell in a layout facility. In particular, once computations have been undergone by the manager holding (a set) of these cells, the offset and size attributes should represent the (window coordinate) space that this cell takes up, on one dimension. (Two cells are needed for two dimensions.)

weight = None¶

A floating point number indicating how much space this cell should be given relative to other cells in the same layout.

fixed = None¶

A numeric amount of fixed space this cell must contain (measured in the window coordinate system).

offset = None¶

The offset from the origin on the dimension specifying this cell (-1 if not computed or not in a layout).

size = None¶

The size of this cell on this dimension (-1 if not computed or not in a layout).

class layout.LayoutVector(*cells)[source]¶

Bases: object

A LayoutVector contains a vector (well, a list) of cells arranged along one dimension. The layout may be computed against a dimensional measure (in any coordinate system, though typically the window coordinate system is used for compatibility with the rest of this module), and the LayoutCell.offset and LayoutCell.size attributes will be set appropriately.

The constructor may be given either one parameter (the number of LayoutCells to construct), or an arbitrary number of positional parameters containing LayoutCells or derivatives thereof (or, really, any object implementing that interface).

Furthermore, __getitem__, __len__, and __iter__ are implemented, and defer calls to the underlying cell collection.

Compute(dim)[source]¶

Compute the layout of the cells along this dimension, assuming a size along this dimensions of the argument given.

class layout.Widget(xcell, ycell, fcol=None, bcol=None, **kwargs)[source]¶

Bases: scenegraph.Renderable

A Widget is a special type of scenegraph.Renderable that expects to be in an environment where:

  • The projection and modelview matrices are identity, giving a precise mapping between input (vertex) coordinates and normalized device coordinates, but
  • the viewport is set so that the rendering bounds are only a small portion of the window–specified by the cell pair given.

As such, the scenegraph.Renderable.PushState() and scenegraph.Renderable.PopState() functions are overriden (such that the scenegraph.Renderable.enable, scenegraph.Renderable.disable, and scenegraph.Renderable.modifications will not work). Their primary job is to set up the viewport, as specified above. (The matrices are assumed to be identity beforehand, an assumption usually guaranteed by an enclosing Container).

The xcell and ycell parameters should be set to a cell on corresponding layout axes.

xcell = None¶

A LayoutCell along the x axis.

ycell = None¶

A LayoutCell along the y axis.

fcol = None¶

A vmath.Vector cotaining the foreground color, or None (whose application differs per widget).

bcol = None¶

A vmath.Vector containing the background color, or None (whose application differs per widget).

pos[source]¶

A 2D vmath.Vector containing the cell positions.

size[source]¶

A 2D vmath.Vector containing the cell sizes.

Note

This does not respect any of the other scenegraph.Renderable attributes, including scenegraph.Renderable.enable, scenegraph.Renderable.disable, scenegraph.Renderable.modifications, and so on.

class layout.Container(grid, xcell=None, ycell=None, **kwargs)[source]¶

Bases: layout.Widget

A Container is a Widget that contains other Widgets. In particular, a Container is special in that it can accept an Widget.xcell and Widget.ycell value of None, implying that this container is the top-level container, which causes it to do useful duties (like set up the identity matrices; see Widget).

Containers always have a layout system–presently, grid (though the name is subject to change). It is logical (but not required) to put Widgets that are in this layout system as children of the Container. Other scenegraph.Renderables should not be made children of Containers due to the odd circumstances under which Widgets are rendered.

PushState()[source]¶

Initialize the state. Depending on whether this is a top-level container, this may initialize the matrices (without affecting the viewport), or it may just set a viewport as with the usual Widget.PushState().

PopState()[source]¶

Reverts the state, undoing the actions done during PushState().

Render()[source]¶

Returns a tuple (:class:`LayoutCell`, :class:`LayoutCell`) that represents the cell pair at the given position in the layout held by this Container. This may be used for various sorts of hit-testing.

ChildAt(pos)[source]¶

Returns a Widget at the position specified, if one exists there; otherwise, returns None.

TriggerChildren(ev)[source]¶

Overrides the default propagation behavior by ensuring that Event objects with a pos attribute are dispatched only to Widget objects in that position.

Note

This special naming convention is all-inclusive; any Event with a pos attribute will be propagated thusly, and any without a pos attribute will simply propagate to all children, as usual. This means you can use the same behavior in your position-sensitive events, as well.

class layout.Label(xcell, ycell, text='', align=0, font=None, **kwargs)[source]¶

Bases: layout.Widget

The Label is a Widget designed to display text.

Since this function is useful to some other Widgets, it’s also the base class for a few.

text = None¶

A string containing the text to display.

align = None¶

The alignment mode (a ALIGN bit mask).

font = None¶

The pygame.Font object to use for rendering.

tex = None¶

The scenegraph.Texture used to store the font texture.

Update(text=None)[source]¶

Updates the Label, drawing the new text to the tex scenegraph.Texture.

Note

This is done automatically whenever text is no longer determined to be the same object as before. Since strings in Python are immutable, this must be the case if the value is no longer the same as well. However, this process is not performed if the Widget.fcol attribute is changed (which determines the font color), and so it must be either called manually from code that does this, or the text attribute must be changed to a different object. A trivial way of performing the latter is assigning label.text=str(label.text).

This behavior may change in later versions to update when the color is changed as well.

Render()[source]¶

Renders the label using the current viewport.

RenderText()[source]¶

Renders the text–a process which is usable by subclasses as needed.

Warning

While GL_ENABLE_BIT is pushed before rendering, GL_TEXTURE_2D is still generally enabled after this process. You’ll want to disable it before doing your own rendering after calling this, as the 0, 0 texel generally has an alpha value of 0.

This is a known bug and will be fixed.

A Slider is a type of widget that allows a user to manipulate a value by providing an axis between two extreme values; the user is expected to use the mouse to select the value as a point within this range; the value is readable as the value attribute.

The orientation (a ORIENT constant).

The floating-point width of the slide handle in normalized device coordinates (where 1 would be half the viewport).

hcol = None¶

A 4D vmath.Vector color of the slide handle (or None for the default transparent gray).

range[source]¶

The computed difference between max and min.

ratio[source]¶

The position of value normalized such that the minimal point is 0 and the maximal point is 1.

static Step(n)[source]¶

Produces a lambda function that may be used as a value for mapfunc. The return function will produce values x such that x*n is an integer (barring round-off error), or, alternatively, will clamp values to 1/n increments.

Calling this with n==1 will result in an analogue to the int function, which should probably be used instead.
