event – Events¶

This module defines events that may be used anywhere that an event-driven system is required.

class event.EVENT[source]¶

An enumeration containing valid values for Event.type.

KBD = 1¶

An event relating to the keyboard (key down, key up, character, ...)

MOUSE = 2¶

An event relating to the mouse (button press, release, mouse move, ...)

class event.KBD[source]¶

An enumeration containing event subtypes for the EVENT.KBD event.

KEYDOWN = 1¶

A key was pressed.

KEYUP = 2¶

A key was released.

CHAR = 3¶

A printable character was generated

Note

Printable character generation may occur multiple times between a key press and key release (such as when the key is held for long enough to start auto-repeating), or not at all (such as if the key is a non-printable, like a modifier key or an F-key). In general, if you’re interested in what the user is typing, hook CHAR; otherwise, if you’re interested in the state of the keyboard, hook KEYDOWN and KEYUP.

class event.MOUSE¶

An enumeration containing event subtypes for the EVENT.MOUSE event.

BUTTONDOWN = 1¶

A mouse button was pressed.

BUTTONUP = 2¶

A mouse button was released.

WHEEL = 3¶

The mouse wheel was scrolled (or pivoted on some axis).

Note

Due to the way SDL handles the scroll wheel, you may never see this event. To my knowledge, however, some persons patched the SDL code to interpret scroll wheel up and scroll wheel down movements as buttons 4 and 5–this is unfortunate for persons (like myself) that have 5-button mice and a scroll wheel that can also support horizontal movement.

Barring this poor design decision, the engine will try to provide the most accurate representation of what really happened, even if SDL is coaxed into lying about it.

MOVE = 4¶

The cursor was moved.

class event.Event(type, **kwargs)[source]¶

Bases: object

An Event contains all of the information needed to pass an event to a system which uses events.

Events may be given arbitrary attributes from the keyword arguments passed to the constructor. A later type=... in the keyword arguments will override the specified event type given as a positional parameter, if one is provided.

type = None¶

The event type (one of the EVENT values). Other attributes depend on the event.
